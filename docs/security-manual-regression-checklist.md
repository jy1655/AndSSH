# 보안 수동 회귀 체크리스트

기준: 릴리스 전/보안 변경 후 1회 실행

## A. 앱 잠금 / 생체 인증

- [ ] 생체 인증 활성화 상태에서 앱 잠금 진입 시 생체 팝업이 자동으로 표시된다.
- [ ] 생체 인증 취소 후 비밀번호 입력으로 정상 해제된다.
- [ ] 잘못된 비밀번호 입력 시 오류 문구가 표시되고 잠금이 유지된다.
- [ ] 자동 잠금 타임아웃(예: 30초) 경과 후 복귀 시 잠금이 다시 걸린다.

## B. 클립보드 보호

- [ ] 터미널 복사 시 클립보드에 텍스트가 저장된다.
- [ ] 설정한 자동 삭제 시간 경과 후 클립보드가 비워진다.
- [ ] Android 13+에서 민감 플래그(`EXTRA_IS_SENSITIVE`)가 적용된다.

## C. 화면 보호

- [ ] `Settings > Screenshot Protection` ON 시 스크린샷/최근앱 미리보기가 차단된다.
- [ ] 같은 설정 OFF 시 원래 동작으로 복원된다.
- [ ] 설정 토글 후 재시작 없이 즉시 반영된다.

## D. 보안키 SSH 경로 상태 검증 (현재 `main`에서는 N/A)

- [ ] README, 릴리스 노트, 인앱 문구가 현재 릴리스에서 보안키 SSH 인증이 불가능하다는 사실과 모순되지 않는다.
- [ ] UI 어디에도 제거된 `Enroll Security Key` / `Copy authorized_key` 진입점이 남아 있지 않다.
- [ ] 보안키 관련 검증이 필요하면 현재 브랜치의 실제 코드에 맞게 `docs/security-key-e2e.md`를 먼저 갱신한 뒤 진행한다.
- [ ] 백업/복원 또는 연결 호환성 문구가 legacy security-key continuity를 암묵적으로 약속하지 않는다.
